php7和一些扩展的安装配置

发表于 2016.01.03   |   分类于 代码与诗

15年底最后一天接受了这个任务，要在项目第N次上线前，生产环境升级成php7的版本。断断续续用了2个工作日，在测试服务器上调试，差不多解决了安装问题，项目代码的一些地雷也检测出，不排除还有些雷没碰到。

安装php7

下载
下载页面： http://php.net/archive/2015.php#id2015-01-11-6

编译安装
基本的组件可以事先通过yum安装好

./configure --prefix=/usr/local/php7 --with-config-file-path=/usr/local/php7/etc --enable-fpm --with-fpm-user=www --with-fpm-group=www --with-mysqli --with-pdo-mysql --with-iconv-dir --with-freetype-dir --with-jpeg-dir --with-png-dir --with-zlib --with-libxml-dir=/usr --enable-xml --disable-rpath --enable-bcmath --enable-shmop --enable-sysvsem --enable-inline-optimization --with-curl --enable-mbregex --enable-mbstring --with-mcrypt --enable-ftp --with-gd --enable-gd-native-ttf --with-openssl --with-mhash --enable-pcntl --enable-sockets --with-xmlrpc --enable-zip --enable-soap --without-pear --with-gettext --disable-fileinfo --enable-maintainer-zts
make && make install

配置
从源目录复制默认配置文件到指定目录

cp php.ini-production /usr/local/php7/etc/php.ini
cp sapi/fpm/init.d.php-fpm /etc/init.d/php7-fpm
chmod +x /etc/init.d/php7-fpm
cp /usr/local/php7/etc/php-fpm.conf.default /usr/local/php7/etc/php-fpm.conf
cp /usr/local/php7/etc/php-fpm.d/www.conf.default /usr/local/php7/etc/php-fpm.d/www.conf

修改配置文件/usr/local/php7/etc/php-fpm.d/www.conf

listen = /dev/shm/php.socket
listen.backlog = 16384
pm = static
pm.max_children = 128
pm.max_requests = 204800
slowlog = var/log/slow.log
request_slowlog_timeout = 1s
rlimit_files = 102400

以上是三国项目中的配置

启动、停止、重启
两种方式

/etc/init.d/php7-fpm start/stop/restart

或

service php7-fpm start/stop/restart

安装redis、memcache扩展

memcached

php的memcache有两个扩展，memcached和libmemcached。libmemcached是php的原生驱动，速度更快，且支持最新的php7，所以原来用的memcached扩展废弃，重新安装libmemcached，需要先安装libevent组件。

安装libmemcached

wget http://launchpad.net/libmemcached/1.0/0.43/+download/libmemcached-0.43.tar.gz (https://launchpad.net/libmemcached/+download)
tar xvzf libmemcached-0.38.tar.gz
cd libmemcached-0.38
./configure --prefix=/usr/local/libmemcached/ --with-libmemcached-dir=/usr/local/libmemcached/ --with-memcached=/usr/local/memcached/bin/memcached
make
make install

安装php扩展

git clone https://github.com/php-memcached-dev/php-memcached.git
cd php-memcached
/usr/local/php7/bin/phpize
./configure  -with-php-config=/usr/local/php7/bin/php-config -with-libmemcached-dir=/usr/local/libmemcached/ --disable-memcached-sasl
make
make install

redis

redis的php扩展有好几个版本，最标准的自然就是redis官网上推荐的那个。不过与我使用的https://github.com/phpredis这个版本代码差异较大。鸟哥还推荐过一个版本，经测试，这版的redis扩展对gzcompress压缩支持的不完善，压缩后的数据存入redis再取出就会发生错误。所以还是用了现有版本的php7分支。

php7分支下载： https://github.com/phpredis/phpredis/tree/php7

下载解压后进入源码目录

/usr/local/php7/bin/phpize
./configure  -with-php-config=/usr/local/php7/bin/php-config
make
make install

代码修改

之前使用了memcache类和mysql驱动，相应的改成new Memcached()和Mysqli，memcached的连接方式也有所改变。

php7里面对象的属性名也有所变化，可以使用变量，却不能再使用数组元素。
